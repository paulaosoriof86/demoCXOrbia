# Pendientes prototipo - Academia por roles

Fecha: 2026-07-04

## Decision

Academia debe profundizarse como modulo de capacitacion y adopcion por roles.

## Pendientes para prototipo

1. Rutas de aprendizaje por rol:
   - superadmin;
   - admin;
   - operativo/coordinador;
   - shopper;
   - cliente final;
   - consultora/representante/franquiciado/aliado/socio.

2. Cada ruta debe incluir:
   - induccion;
   - modulos que usa;
   - operacion paso a paso;
   - botones y acciones;
   - ventajas;
   - errores comunes;
   - checklist de validacion;
   - opcion de solicitar capacitacion por tema.

3. Shopper debe tener profesionalizacion sectorial:
   - que es mystery shopping;
   - que es escenario;
   - como se llena un cuestionario;
   - como justificar hallazgos negativos;
   - evidencias;
   - beneficios y liquidaciones.

4. Cliente y consultora deben tener contenido comercial:
   - beneficios de la plataforma;
   - modulos disponibles;
   - paquetes adicionales;
   - automatizaciones e integraciones;
   - como solicitar servicios nuevos.

5. Cada modulo revisado por backend debe generar pendiente Academia asociado.
