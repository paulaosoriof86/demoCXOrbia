# PENDIENTES PROTOTIPO — materialización Firestore R6

1. Si se muestra readiness/importación, distinguir `plan_prepared`, `validated`, `pending_authorization`, `materializing`, `confirmed` y `failed`.
2. Mostrar conteos por dominio sin presentar filas privadas.
3. Materialización debe permanecer desactivada mientras el gate esté HOLD.
4. No afirmar que Firestore está conectado ni que los datos fueron importados.
5. No asociar pago o certificación a materialización mientras sus fuentes sigan pendientes.
6. Academia/manuales deben explicar plan vs ejecución real.
