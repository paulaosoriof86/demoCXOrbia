# RESUMEN PARA CLAUDE - addendum forensic audit readiness

Fecha: 2026-07-09

## Instruccion para futura auditoria de candidata

Cuando llegue la nueva candidata, no debe aprobarse solo por apariencia. Debe compararse contra la candidata actual y contra todo lo solicitado en los paquetes acumulados.

## Debe verificarse

- Proyecto separado de periodo.
- Tenant/proyecto configurable.
- HR/source enmascarado y generico.
- Roles/personas/scopes configurables.
- Auth/gates/datos protegidos honestos.
- Academia, cursos y manuales por rol.
- Certificaciones carryover.
- Liquidaciones/pagos como preview/review sin ejecucion real.
- ReviewQueue y auditEvents.
- Branding/PWA dinamico.
- Make/Gemini/pagos gate-off.
- Separacion TyA especifico vs reusable CXOrbia.

## Salida esperada para Claude

Al auditar, clasificar pendientes en:

- P0 para pedir ahora.
- P1 para pedir si conserva capacidad.
- Backend ya preparado.
- Prototipo/Claude requerido.
- Configuracion TyA.
- Patron reusable CXOrbia.
