# Pendientes prototipo addendum role taxonomy org scope

Fecha: 2026-07-09

## Pendientes para prototipo/Claude

1. Mostrar personas operativas configurables, no solo roles tecnicos.
2. Incluir representantes, coordinadores, franquiciados/franquicia, cliente/marca evaluada, finanzas, certificaciones y shopper/evaluador.
3. Permitir scope por tenant, pais, proyecto y propio perfil.
4. Mantener claims como contrato/gate si Auth no esta activo.
5. Mostrar estados honestos: pendiente Auth, gate apagado, requiere acceso.
6. No hardcodear TyA ni una estructura unica.
7. No exponer datos sensibles ni emails reales en fixtures.
8. Actualizar Academia con persona vs rol tecnico.

## Criterio de NO GO

NO GO si el prototipo sigue mostrando solo admin/shopper como estructura de permisos, si no permite representantes/coordinadores/franquicia/cliente, o si mezcla permisos sin scope por tenant/proyecto/pais.
