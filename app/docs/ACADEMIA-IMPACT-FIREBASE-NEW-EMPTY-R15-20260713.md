# ACADEMIA — IMPACTO R15 Firebase DEV

- DEV nuevo y vacío es un entorno técnico, no producción.
- Proyecto Firebase creado no significa que Auth, Firestore, Storage, Make, Gemini, pagos o sincronización estén activos.
- Cada activación posterior requiere gate, autorización, rollback y auditoría.
- No mostrar identificadores de credenciales ni datos de proveedor en manuales/cursos.
