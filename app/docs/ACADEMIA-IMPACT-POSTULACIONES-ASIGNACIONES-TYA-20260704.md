# Academia impact - Postulaciones y asignaciones TyA

Fecha: 2026-07-04

## Objetivo

Aplicar el gate de Academia al primer bloque backend modulo por modulo: postulaciones y asignaciones.

## Manuales a actualizar o crear

1. Manual de postulaciones para admin/ops.
2. Manual de asignaciones para admin/ops.
3. Manual de postulacion para shopper.
4. Manual de conflictos HR/plataforma.
5. Manual de estados de sincronizacion.

## Cursos relacionados

### Operativo / coordinador

Curso: Gestion diaria de postulaciones y asignaciones.

Lecciones:

1. Que es una postulacion.
2. Como revisar una postulacion.
3. Como aprobar o rechazar.
4. Como asignar manualmente una visita.
5. Como confirmar que una visita salio de disponibles.
6. Como interpretar estado pendiente de sync.
7. Como escalar conflicto HR/plataforma.
8. Errores comunes.

### Admin

Curso: Control operativo de asignaciones.

Lecciones:

1. Diferencia entre postulacion, asignacion y visita disponible.
2. Origen de asignacion: plataforma vs HR.
3. Estados de sincronizacion.
4. Revision de conflictos.
5. Notificaciones al shopper.
6. Validacion de cobertura.

### Shopper

Curso: Como postularme y gestionar visitas asignadas.

Lecciones:

1. Como encontrar visitas disponibles.
2. Como postularme.
3. Que pasa cuando me aprueban.
4. Como veo una visita asignada.
5. Que notificaciones recibo.
6. Que hacer si necesito reprogramar o cancelar.

### Cliente final

Curso: Como leer avance de cobertura.

Lecciones:

1. Que significa visita disponible.
2. Que significa visita asignada.
3. Que significa visita en proceso.
4. Como interpretar cobertura sin ver datos internos sensibles.

### Consultora / representante / franquiciado / aliado / socio

Curso: Valor operativo y comercial de la sincronizacion.

Lecciones:

1. Como la plataforma evita duplicidades.
2. Por que HR/plataforma deben sincronizarse.
3. Que valor aporta al cliente.
4. Que beneficios tienen automatizaciones y paquetes adicionales.

## Solicitudes de capacitacion disponibles

- Necesito aprender a aprobar postulaciones.
- Necesito aprender a asignar visitas.
- Necesito aprender a resolver conflictos HR/plataforma.
- Necesito aprender los estados de sync.
- Necesito aprender como shopper a postularme.

## Checklist interactivo recomendado

Para admin/ops:

- Revise postulacion.
- Valide shopper.
- Valide visita/proyecto.
- Apruebe o rechace.
- Verifique que la visita salio de disponibles.
- Verifique estado de sync.
- Registre o escale conflicto si aplica.

Para shopper:

- Revise visita disponible.
- Lea instructivo.
- Postulese.
- Espere aprobacion.
- Revise notificacion.
- Agende segun reglas.
