# Pendientes prototipo - Restricciones y ficha de postulacion Phase A

Fecha: 2026-07-04

## Pendientes visuales derivados del contrato backend

1. La ficha de postulacion debe mostrar:
   - honorario;
   - reembolso;
   - escenario;
   - resumen del proyecto;
   - restricciones de perfil;
   - restricciones de frecuencia y ultima visita;
   - restricciones de dias, horario, franja o ventana;
   - certificacion requerida;
   - documentos requeridos;
   - evidencias requeridas;
   - tipo de cuestionario;
   - si la restriccion es requisito del cliente o medicion interna.

2. Cinepolis debe mantener WK/WKND y quincena como configuracion del proyecto, no como regla global.

3. Otros proyectos deben poder configurar restricciones propias como edad, frecuencia, ultima visita, horario, dias, zona, experiencia, documentos o certificacion.

4. Si la elegibilidad es dudosa, debe existir revision manual.

5. No debe mostrarse HR sincronizada si el gate esta apagado.

## Pendientes Academia asociados

- Curso shopper: como leer ficha de postulacion.
- Curso ops: validar restricciones.
- Curso admin: configurar restricciones por proyecto.
- Curso cliente: requisitos de visita.
- Glosario: elegibilidad, restriccion de perfil, requisito del cliente, medicion interna, ultima visita, frecuencia.

## Pendiente futuro

Crear contrato dedicado de scoring/ranking shopper antes de aplicar penalizaciones automaticas.
