# Pendientes prototipo - Email provider agnostic y liquidaciones

Fecha: 2026-07-04

## Email por usuario y cualquier proveedor

- Configurar correo por usuario.
- Elegir proveedor: Gmail/Google Workspace, Microsoft 365/Outlook, IMAP/SMTP generico, SMTP custom, API custom futura, manual log only o desconocido por configurar.
- Permitir correo personal, asignado por TyA, asignado por consultora, creado por CXOrbia futuro, alias de rol, placeholder sin envio o contacto manual.
- Crear usuario con o sin correo.
- Mostrar estado real: no configurado, manual, placeholder, OAuth futuro, SMTP futuro, conectado futuro, error futuro, revocado o deshabilitado.

## Liquidaciones y pagos

- Separar visita realizada, cuestionario realizado, submitido, liquidacion y pago.
- Mis beneficios debe mostrar honorario, reembolso, pendiente, programado y pagado.
- Admin debe revisar corte/lote.
- Junio ejecutado debe entrar como pago pendiente, no visita pendiente.
- No exponer banco/DPI/datos sensibles.
- No marcar pagado sin confirmacion real/manual futura.

## Academia

- Manual correo por usuario/proveedor.
- Curso admin de creacion de usuarios con correo.
- Manual shopper de Mis beneficios.
- Curso admin de liquidaciones y cortes.
- Glosario financiero y de correo.
