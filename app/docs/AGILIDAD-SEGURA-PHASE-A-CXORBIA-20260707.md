# Agilidad segura Phase A - CXOrbia

Fecha: 2026-07-07

## Bloque completado

Se documento una forma de avanzar mas rapido sin subir riesgo.

## Regla central

Acelerar solo lo que es reversible, documentado y sin efecto real externo.

No subir de riesgo hasta tener gates, source lock, auditoria y decision explicita.

## Carriles

### Carril seguro

Avanzar en paralelo con:

- contratos;
- documentos;
- matrices;
- paquetes Claude;
- prompts de continuidad;
- readiness;
- validadores preview-only.

### Carril con gate

Solo avanzar con validacion:

- candidato frontend;
- empalme de prototipo;
- cambios visibles;
- staging;
- smoke visual;
- smoke remoto.

### Carril bloqueado

No activar todavia:

- proveedores reales;
- escritura real;
- import real;
- sync real;
- salida real.

## Como agilizar

1. Cerrar bloques seguros por lotes.
2. No regenerar paquete Claude despues de cada microbloque.
3. Regenerar paquete completo al cierre de lote.
4. Enviar a Claude paquete completo con prioridades.
5. Mantener UI sin tocar hasta source lock o candidata de Claude.
6. Usar gates antes de cada subida de riesgo.
7. Documentar bloqueos sin afirmar que quedaron hechos.

## Anti-inestabilidad

- no tocar UI sin source lock;
- no empalmar sin auditoria;
- no conectar base vieja;
- no ocultar conflictos;
- no hacer acciones silenciosas desde dashboard;
- no subir de carril sin gate.
